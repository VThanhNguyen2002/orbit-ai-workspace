# ADR-010: Embedding And Vector Search

**Status:** Accepted
**Date:** 2026-05-19
**Author:** Viet Thanh Nguyen

## Context

Synapse needs semantic search over personal knowledge so users can find notes and transcripts by meaning, not exact keywords. The solution must respect user isolation, control AI cost, and remain simple enough for Phase 2.

## Decision

Use **Supabase Postgres with pgvector** for embeddings and vector search.

Phase 2 indexes text sources generated by notes and transcripts. Content is chunked, embedded by the backend, stored with `user_id`, source metadata, model name, and chunk index, then queried with cosine similarity.

## Rationale

- pgvector keeps vector search inside the existing Postgres/Supabase data boundary.
- RLS can apply to embedding rows like every other user-owned table.
- No separate vector database is required for Phase 2 scale.
- Storing model name and chunk index supports future re-embedding and result debugging.

## Consequences

### Positive

- One persistence system for relational data and semantic index metadata.
- Search results can join back to source entities with normal SQL.
- User isolation remains enforceable through RLS.
- Operational complexity stays low.

### Negative

- pgvector performance must be watched as embedding counts grow.
- Changing embedding models requires re-embedding content.
- Provider costs scale with content updates and search volume.

### Mitigations

- Chunk long content with stable chunk ordering.
- Record embedding model and source content timestamps.
- Regenerate embeddings only when source text changes.
- Add indexes appropriate for the selected vector dimension and expected dataset size.
- Track query embedding time and search time in response metadata/logs.

## Implementation Notes

Search is backend-only:

1. Client sends a text query.
2. Backend validates and embeds the query.
3. Postgres filters by `user_id` and ranks by vector distance.
4. API returns source metadata, snippet, similarity, and timing metadata.

Do not call embedding providers from the client.
